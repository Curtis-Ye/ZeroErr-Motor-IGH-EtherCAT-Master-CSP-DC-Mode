# 主函数解析5——主控制代码
主控制代码是这份代码里的第二个while(1)循环，本篇分析这个循环内的内容。
```
int32_t actPos0, targetPos0;
int32_t actVel0, targetVel0;
```
这两行代码定义了4个int32_t类型的变量，看名知意，可以清楚地知道这几个变量的含义分别是实际位置、目标位置、实际速度、目标速度。  
```
#ifdef MEASURE_PERF
/* The slave time received in the current and the previous cycle */
uint32_t t_cur, t_prev;
#endif
```
如果定义了`MEASURE_PERF`，则定义类型为uint32_t的两个变量t_cur以及t_prev。 
   
`struct timespec sleepTime;`
定义一个timespec类型的结构体sleepTime。  
```
#ifdef MEASURE_TIMING
	struct timespec execTime, endTime;
#endif
```
和上面相同的解释，这里不过多介绍。  
  
`sleepTime = cycleTime;`  
令sleepTime的值等于cycleTime。  
`clock_gettime(CLOCK_MONOTONIC, &wakeupTime);`  
更新唤醒时间等于当前系统时间。  
****
以上部分是while(1)循环的铺垫，下面进行循环内部代码的分析。  
```
#ifdef MEASURE_TIMING
	clock_gettime(CLOCK_MONOTONIC, &endTime);
	/* wakeupTime is also start time of the loop. */
	/* execTime = endTime - wakeupTime */
	timespec_sub(&execTime, &endTime, &wakeupTime);
	printf("Execution time: %lu ns\n", execTime.tv_nsec);
#endif
```
同样先进行预定义判断，第一个函数clock_gettime()的作用是令endTime的值等于系统时间；第二个函数是将endTime与wakeupTime作差；最后打印信息。  
`timespec_add(&wakeupTime, &wakeupTime, &sleepTime);`  
用于将wakeupTime与sleepTime相加。  
`clock_nanosleep(CLOCK_MONOTONIC, TIMER_ABSTIME, &wakeupTime, NULL);`  
用于延时，之前的分析文件有提及。  
`ecrt_master_receive(master);`  
该函数用于从网卡接收 EtherCAT 从站返回的数据，并更新到主站内存。如果不调用它，读到的 PDO 数据很可能一直是旧数据。  
`ecrt_domain_process(domain1);`  
该函数是 IGH EtherCAT 主站中处理接收到的域（Domain）数据的函数。把刚收到的 EtherCAT 数据包解析出来，并更新 Domain 中的 PDO 数据区。
```
#ifdef MEASURE_PERF
	ecrt_master_reference_clock_time(master, &t_cur);
#endif
```
该函数是 IGH EtherCAT 主站提供的一个读取参考时钟（Reference Clock）当前时间值的函数。  
```
actPos0 = EC_READ_S32(domain1_pd + actual_position);
actVel0 = EC_READ_S32(domain1_pd + actual_velocity);
printf("actPos0: %d \t\t", actPos0);
printf("actVel0: %d", actVel0);
```
这个代码块用于读取实际位置与实际速度。使用EC_READ_S32函数进行读取。这个函数的参数是一个指针，前面有提到domain1_pd是一个指向domain1的指针，其指向domain1的第一个对象。加上actual_position相当于这个指针前进（后退）了该整数值个位置，这个位置存放的便是实际位置的值。读取后通过printf函数打印。  
`targetPos0 = actPos0 + 5000;`
修改目标位置值，每个循环加5000。这里5000是指关节模组编码器数值。但是实际上在官方的代码中这部分内容并没有使用到，因为官方是CSV同步周期速度模式。  
```
uint16_t statusWord = EC_READ_U16(domain1_pd + statusword);
uint16_t state = getDriveState(statusWord);
uint16_t cw = 0; // 将变量声明移到switch语句之前
```
这里的EC_READ_U16函数同样用于读取PDO数据，这里用于读取状态字。第二行用于获取关节模组状态，getDriveState函数用于“加工”statusWord，将返回值存放在state变量里。第三行定义一个cw变量并赋值0。  
```
switch (state)
{
	case STATE_FAULT:
		cw = CONTROL_WORD_FAULT_RESET;
		printf("Fault state, sending reset command\n");
		break;
	case STATE_SWITCH_ON_DISABLED:
		cw = CONTROL_WORD_SHUTDOWN;
		printf("Switch on disabled, sending shutdown command\n");
		break;
	case STATE_READY_TO_SWITCH_ON:
		cw = CONTROL_WORD_SWITCH_ON;
		printf("Ready to switch on, sending switch on command\n");
		break;
	case STATE_SWITCHED_ON:
		cw = CONTROL_WORD_ENABLE_OPERATION;
		printf("Switched on, sending enable operation command\n");
		break;
	case STATE_OPERATION_ENABLED:
		// CSV mode operation
		actVel0 = EC_READ_S32(domain1_pd + actual_velocity);
		// Set constant target velocity
		EC_WRITE_S32(domain1_pd + target_velocity, 10000);
		// Keep operation enabled
		cw = CONTROL_WORD_ENABLE_OPERATION;
		printf("Velocity: Target=10000, Actual=%d\n", actVel0);
		break;
	default:
		cw = CONTROL_WORD_SHUTDOWN;
		printf("Unknown state (0x%04x), trying shutdown\n", state);
		break;
}
EC_WRITE_U16(domain1_pd + controlword, cw);
ecrt_domain_queue(domain1);
```
这个switch语句用于判断当前关节模组的状态，判断完一次状态后赋值给cw变量，break跳出switch循环再将cw变量写入控制字。ecrt_domain_queue用于把 Domain 中修改过的 PDO 数据加入到下一次发送的 EtherCAT 报文中。  
```
#ifdef SYNC_REF_TO_MASTER
	/* Distributed clocks */
	clock_gettime(CLOCK_MONOTONIC, &time);
	ecrt_master_application_time(master, TIMESPEC2NS(time));
	ecrt_master_sync_reference_clock(master);
	ecrt_master_sync_slave_clocks(master);
#endif
```
这个代码块与之前提到的一致，用于更新系统时钟。  
`ecrt_master_send(master);`  
这个函数用于把之前准备好的所有 EtherCAT 数据报（Datagram）封装成 EtherCAT 帧，通过网卡真正发送到 EtherCAT 网络中。 
``` 
#ifdef MEASURE_PERF
	printf("\nTimestamp diff: %" PRIu32 " ns\n\n", t_cur - t_prev);
	t_prev = t_cur;
#endif
```
用于计算时间差，但是在文件中并没有明确体现。  
`return 0;`  
最后假如没有进入while(1)循环，则退出，返回值为0。一般不会有这种情况。
