# 主函数解析2
```
master = ecrt_request_master(0); // 请求主站
if (!master)
	printf("Requesting master failed\n");
```
该代码块的作用是请求主站，可以理解为“打开一个主站”。  
`initDrive(master, 0);`  
这行代码的作用是初始化关节模组。  
```
uint16_t alias = 0;		    // 从站别名
uint16_t position0 = 0;		    // 位置
uint32_t vendor_id = 0x5a65726f;    // 零售商ID
uint32_t product_code = 0x00029252; // 制造商ID
```
这个代码块的作用是定义关节模组的基本属性。在后续的代码中有用到。  
```
ec_slave_config_t *drive0 = ecrt_master_slave_config(master, alias, position0, vendor_id, product_code); // 从站配置
ec_slave_config_state_t slaveState0;
```
`ecrt_master_slave_config`函数的作用可以理解为创建一个从站，返回一个ec_slave_config_t类型的对象。第二行代码则声明了类型为ec_slave_config_state_t的slaveState0变量。  
```
if (!drive0)
{
	printf("Failed to get slave configuration\n");
	return -1;
}
```
这一个代码块用于判断从站是否被配置。  
`#ifdef CONFIG_PDOS`这个预定义标识符可以去看前几篇的预定义标识符解析，这里不再介绍。  
```
ec_pdo_entry_info_t slave_0_pdo_entries[] =
{
	{0x6040, 0x00, 16}, /* Controlword */
	{0x607a, 0x00, 32}, /* Target Position */
	{0x60FF, 0x00, 32}, /* Target Velocity */
	{0x6071, 0x00, 16}, /* Target Torque */

	{0x6041, 0x00, 16}, /* Statusword */
	{0x6064, 0x00, 32}, /* Position Actual Value */
	{0x606c, 0x00, 32}, /* Velocity Actual Value */
	{0x6077, 0x00, 16}, /* Torque Actual Value */
	//{0x6060, 0x00, 8}, /* Control operation */
};
```
这个代码块用于配置PDO参数，之前也有提到PDO在EtherCAT中用于实时数据交互，这里用来配置PDO要传递哪些参数。比如0x607a是主站高速从站目标位置。值得一提的是前面四行是RxPDO，后四行是TxPDO。这个Rx和Tx是对从站而言，以从站的视角来看。其实这个数组是PDO的子索引。    
```
ec_pdo_info_t slave_0_pdos[] =
{
	{0x1600, 4, slave_0_pdo_entries + 0}, /* 2nd Receive PDO Mapping */
	{0x1a00, 4, slave_0_pdo_entries + 4}, /* 2nd Transmit PDO Mapping */
};
```
这个代码块定义一个ec_pdo_info_t类型的数组。根据官方说明书可知：仅 0x1600 映射 RxPDO 支持任意 mapping 配置，包括但不限于 TxPDO：0x6041、0x6064、0x606C、0x6061 等。  
仅 0x1A00 映射 TxPDO 支持任意 mapping 配置，包括但不限于 RxPDO：0x6040、0x607A、0x6065、0x6060 等。  
**这里也涉及到一个概念mapping**：PDO Mapping 本质上就是“对象字典参数与PDO数据帧之间的对应关系配置”。它决定了 EtherCAT 周期通信到底传输哪些数据。  
数组里的参数也好理解：第一个参数就是mapping对象索引，第二个参数是子索引数量，第三个参数是subIndex子索引对象的地址。第一个+0是因为slave_0_pdo_entries是数组名，这意味着slave_0_pdo_entries是子索引数组的指针且指向数组第一个元素，而前面的子索引数量是4，所以这里的索引对应RxPDO，同样道理，后面的代表TxPDO的索引。  
```
ec_sync_info_t slave_0_syncs[] =
{
	{0, EC_DIR_OUTPUT, 0, NULL, EC_WD_DISABLE},
	{1, EC_DIR_INPUT, 0, NULL, EC_WD_DISABLE},	       // 同步管理器0,1用于保留用途，不用于数据交换
	{2, EC_DIR_OUTPUT, 1, slave_0_pdos + 0, EC_WD_ENABLE}, // 输出过程数据
	{3, EC_DIR_INPUT, 1, slave_0_pdos + 1, EC_WD_DISABLE}, // 输入过程数据
         {0xFF}
};
```
用于使能过程数据交互。
