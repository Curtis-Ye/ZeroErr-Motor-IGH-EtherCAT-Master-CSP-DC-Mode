可以看到用到的预定义标识符非常多。下面对以上代码进行分析。  
## `#define CONFIG_PDOS`
这个一行代码用于启用或禁用PDO映射，若禁用则使用保存在EEPROM中的PDO映射配置。
## `#define DC`
用于启用或禁用DC时钟。这个就比较好理解了。
## 代码块1
```
#ifdef DC  
#define SYNC_REF_TO_MASTER  
#endif
```
和程序中的if判断类似，如果定义了DC（即开启DC时钟），就定义`#define SYNC_REF_TO_MASTER`这行代码。如果启用SYNC_REF_TO_MASTER,意味着使用主站时钟作为参考时钟；否则默认使用从站0的时钟作为参考。 
## 代码块2 
```
#ifdef DC  
#define CONFIG_DC  
#endif
```
与代码块1类似。如果开启DC时钟，就开启对从站DC时钟规范的配置。
## `#define ENCODER_RES 524287`
这个预定义标识符指的是关节模组编码器最大测量范围。转一圈的数值是524287。
## `#define MAX_SAFE_STACK (8 * 1024)`
定义最大安全堆栈值。
## `#define MEASURE_PERF`
用于开启性能测试。测量每个周期参考从机时钟时间戳的差异，并打印结果，其应尽可能接近循环时间。只有在开启DC时钟的情况下有效。
## `#define MEASURE_TIMING`
MEASURE_TIMING变量用于存储计算完成循环所需的时间。
## `#define SET_CPU_AFFINITY`
用于设置CPU亲和性
## `#define NSEC_PER_SEC (1000000000L)`
定义纳秒与秒之间的数值转化值1x10^9，L表示长整形。
## `#define FREQUENCY 1000`
定义频率的值为1000
## `#define PERIOD_NS (NSEC_PER_SEC / FREQUENCY)`
运动循环周期，单位为纳秒。
## 代码块3
```
#ifdef DC
#define TIMESPEC2NS(T) ((uint64_t)(T).tv_sec * NSEC_PER_SEC + (T).tv_nsec)
#endif
```
如果启用DC时钟，定义将 struct timespec 转为纳秒整数的宏。
## 代码块4
```
#ifdef CONFIG_DC
#define SHIFT0 (PERIOD_NS / 2)
#endif
```
如果定义了CONFIG_DC，就定义SHIFT0等于循环周期数值的一半。
## 代码块5
代码块5了解即可，定义了SYNC_MASTER_TO_REF才会使用到代码块5中的内容。**SYNC 是 Synchronization（同步）的缩写，表示分布式时钟同步。REF 是 Reference（参考/基准）的缩写。**
