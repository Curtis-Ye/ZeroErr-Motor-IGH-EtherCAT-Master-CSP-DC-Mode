# 官方例程用到的头文件解析
可以看到代码中包括以下头文件：  
`<ecrt.h>`、`<string.h>`、`<stdio.h>`、`<sys/resource.h>`、`<unistd.h>`、`<sys/types.h>`、`<sys/mman.h>`、`<time.h>`、`<signal.h>`、`<sched.h>`、`<inttypes.h>`。第一眼看过去感觉眼花缭乱，而且不少头文件让人感觉很陌生。现在逐一分析：  
## `<ecrt.h>`
这个头文件是整份程序的核心依赖，负责 EtherCAT主站与从站通信。代码中调用的很多API都是由这个文件提供。包括：ecrt_request_master() 请求主站，ecrt_master_slave_config() 找到从站配置，ecrt_slave_config_pdos() 配置 PDO 映射等。  
**这个头文件就是“真正和 EtherCAT 驱动、从站 PDO、DC 同步打交道”的底层入口。**
## `<string.h>`
这个头文件想必就不陌生了，里面是字符串相关的函数。但是这份代码里没用到字符串相关，相反这份代码里最明确的用法是 stack_prefault() 里的 memset(dummy, 0, MAX_SAFE_STACK)，它的作用是把一块栈内存提前“碰一遍”，配合后面的 mlockall() 尽量避免运行时缺页和换页抖动。
## `<stdio.h>`
这个更熟悉了，这份代码里也是用里面最出名的printf函数打印信息。
## `<sys/resource.h>`
文件头注释写的是“用于 setpriority”，但在 main() 里真正设置实时优先级的是 sched_get_priority_max(SCHED_FIFO) 和 sched_setscheduler()，并没有看到 setpriority() 的调用。也就是说，这个头文件在当前代码中并没有发挥注释里写的那个作用，更像是作者早期想用进程 nice 值，后来改成了更适合实时控制的 FIFO 调度。所以这个头文件可能是多余的。
## `<unistd.h>`和 `<sys/types.h>`
用于进程 ID、pid_t、getpid() 这类系统调用类型。
信号处理函数里先 getpid() 拿当前进程号，再 kill(pid, SIGKILL) 强制结束自己；这里 pid_t 的类型定义来自 sys/types.h，getpid() 则来自 unistd.h。这两个头文件在这里配合使用，是为了让程序在收到 SIGINT 时能够先释放 EtherCAT 主站，再把自己干净地杀掉。
## `<sys/mman.h>`
负责把程序锁进内存，减少实时抖动。main() 里调用了 mlockall(MCL_CURRENT | MCL_FUTURE)，明确要求把当前页和未来分配页都锁定在 RAM 中，避免被交换到磁盘；这对 1 kHz 的循环控制非常重要，因为一旦运行时发生缺页，循环延迟就会突然飙升。代码里 stack_prefault() 又进一步把栈区“提前触碰”一遍，和 mlockall() 是配套策略。
## `<time.h>`
负责所有时间戳、周期调度和绝对睡眠。这里不过多介绍，主要知道这个文件负责系统时间。
## `<signal.h>`
用于注册退出信号和使用信号常量。main()里通过signal(SIGINT, signal_handler)注册了中断处理函数，意味着按 Ctrl+C 时不会直接粗暴退出，而是先执行signal_handler()释放 EtherCAT 主站。处理函数内部又使用了 SIGKILL 这个信号常量来终止进程。这个设计说明作者很在意“退出时把主站资源释放掉”，避免 EtherCAT 主站被异常占用。
## `<sched.h>`
这是实时控制的另一大核心，负责 CPU 亲和性和 FIFO 调度。代码里用了 cpu_set_t、CPU_ZERO()、CPU_SET()、sched_setaffinity()来把进程绑到指定 CPU 核心上；又用 sched_get_priority_max(SCHED_FIFO) 和 sched_setscheduler() 把调度策略切到实时 FIFO。  
可以把这一部分理解成“让控制环尽量固定在某个核上，以最高实时优先级运行，减少被普通任务打断的概率”
## `<inttypes.h>`
专门为格式化输出提供标准整数宏。最后循环里打印 Timestamp diff 时用了 PRIu32，这是 inttypes.h 提供的标准格式化宏，可以让 uint32_t 的打印在不同平台上更稳妥，不必手写 %u 去赌类型宽度。
  
**以上便是所有头文件的内容**  
分析一下大致可以看出本质上，这些头文件可以分为几个功能：最基本的，提供EtherCAT的API接口函数、保障实时性的部分、时间计算部分、进程管理部分以及信息、格式部分。  
**除此之外还要提醒的是和平时用Windows系统开发不同，这份代码使用的是IGH EtherCAT主站方案，需要在Linux系统上开发，没有集成好的IDE而需要手把手编译。**
还有一个问题，这些头文件在哪呢？之前做单片机开发的时候都是直接下载对应库然后在IDE中添加对应路径include即可使用。在本项目中，上述库分为两类：  
第一类是Linux系统自带头文件，像下面这些：  
```
#include <string.h>
#include <stdio.h>
#include <unistd.h>
#include <sys/types.h>
#include <sys/mman.h>
#include <time.h>
#include <signal.h>
#include <sched.h>
#include <inttypes.h>
#include <sys/resource.h>
```
都是标准C库，Ubuntu系统自带。  
像`<ecrt.h>`是安装IGH后才有的。
