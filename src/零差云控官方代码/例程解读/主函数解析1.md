# 主函数解析第一部分
```
#ifdef SET_CPU_AFFINITY
	cpu_set_t set;
	CPU_ZERO(&set);
	CPU_SET(4, &set);
#endif
```
这是一个典型的设置CPU亲和性代码块，使用非动态分配cpu_set_t。  
```
if (sched_setaffinity(0, sizeof(set), &set))
{
	printf("Setting CPU affinity failed!\n");
	return -1;
}
```
sched_setaffinity 是一个用于设置进程或线程的 CPU 亲和性的系统调用，它允许将特定的进程或线程绑定到一个或多个 CPU 核心上运行。  
pid：要设置亲和性的进程ID。0表示当前进程。  
cpusetsize：mask指针指向的数据大小，通常为 sizeof(cpu_set_t)  
mask：指向 cpu_set_t 结构的指针，指定CPU亲和性掩码  
函数执行成功将返回0 。  
```
struct sched_param param = {};
param.sched_priority = sched_get_priority_max(SCHED_FIFO);
printf("Using priority %i.\n", param.sched_priority);
if (sched_setscheduler(0, SCHED_FIFO, &param) == -1)
{
	perror("sched_setscheduler failed\n");
}
```
sched_param是描述调度参数的结构体。sched_get_priority_max用于获取最大优先级。sched_setscheduler()函数用以修改线程的调度策略以及调度参数。  
sched_setscheduler(pid_t pid, int policy, const struct sched_param *param)参数说明：  
`pid`表示要修改/设置调度参数的目标线程。  
`policy`表示目标线程的调度策略。目前linux对于sched_setscheduler()函数支持如下调度策略：  
`SCHED_OTHER`、`SCHED_BATCH`、`SCHED_IDLE`、`SCHED_FIFO`、`SCHED_RR`  
`param`表示传入调度参数（如：优先级）  
```
if (mlockall(MCL_CURRENT | MCL_FUTURE) == -1)

	printf("mlockall failed\n");
	return -1;
}
```
用于把当前页和未来分配页都锁定在 RAM 中。防止翻页发生抖动。
