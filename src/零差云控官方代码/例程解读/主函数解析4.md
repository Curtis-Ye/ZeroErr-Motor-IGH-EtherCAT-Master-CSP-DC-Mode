# 主函数解析4
进入第一个while(1)循环的部分，首先看看这个while(1)循环的内部。  
`timespec_add(&wakeupTime, &wakeupTime, &cycleTime);`  
这是时间值相加的辅助函数。用于计算下次唤醒时间。    
`clock_nanosleep(CLOCK_MONOTONIC, TIMER_ABSTIME, &wakeupTime, NULL);`  
等待到指定绝对时间。clock_nanosleep（）允许调用线程以纳秒精度指定的时间间隔进行睡眠。它的不同之处在于允许调用者选择要测量睡眠间隔的时钟，并且允许将睡眠间隔指定为绝对值或相对值。  
函数原型：  
```
   INT clock_nanosleep（clockid_t clock_id ，INT flags，
                       const struct timespec *request,
                       struct timespec *remain）; 

```
clock_id参数具有以下值之一：  
CLOCK_REALTIME 可设置的系统范围的实时时钟。  
CLOCK_MONOTONIC 一个不可设置的，单调递增的时钟，用于测量从过去某个未指定的点开始的时间，该时间在系统启动后不会更改。  
CLOCK_PROCESS_CPUTIME_ID 可设置的每个进程时钟，用于测量进程中所有线程消耗的CPU时间。  
如果flags为0，则将request中指定的值解释为相对于clock_id指定的时钟的当前值的间隔。如果flags是TIMER_ABSTIME，那么request将被解释为由时钟clock_id测量的绝对时间。  
`ecrt_master_receive(master);`  
该函数用于从网卡接收 EtherCAT 从站返回的数据，并更新到主站内存。如果不调用它，读到的 PDO 数据很可能一直是旧数据。
`ecrt_slave_config_state(drive0, &slaveState0);`  
查询从站配置对象（Slave Configuration）的状态信息。  
```
if (slaveState0.operational)
{
	printf("All slaves have reached OP state\n");
	// initDrive(master, 0);
	break;
}
```
这个代码块用于判断drive0的状态信息，并进行判断：如果运行状态正常则打印信息并退出while(1)循环进入主循环。  
`ecrt_domain_queue(domain1);`  
用于把 Domain 中修改过的 PDO 数据加入到下一次发送的 EtherCAT 报文中。  
后续代码我们只关注`#ifdef SYNC_REF_TO_MASTER`之后的代码块。因为我们之前用的参考时钟是主站的时钟，因此只需要关注`#ifdef SYNC_REF_TO_MASTER`内部的代码。  
```
clock_gettime(CLOCK_MONOTONIC, &time);
ecrt_master_application_time(master, TIMESPEC2NS(time));
ecrt_master_sync_reference_clock(master);
ecrt_master_sync_slave_clocks(master);
```
上面四行代码的作用分别是：  
获取PC当前时间（当前CPU时间）、通知EtherCAT主站当前时间、同步参考从站，告诉参考从站其时间应该等于PC时间、同步其他从站。最终将形成：  
PC
↓
Reference Slave
↓
All Slaves  
`ecrt_master_send(master);`  
发送EtherCAT报文,真正把之前准备好的报文发出去。包括DC同步命令，状态机命令。  
为什么需要这个while(1)循环？因为 EtherCAT 从站刚上电时不会立即进入 OP。这个循环不负责运动控制，只负责让网络从启动状态稳定进入运行状态。可以理解为启动汽车前的各项检查与配置。
